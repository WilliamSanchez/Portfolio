FPGA

(2013)

Objetivo

Desenvolvimento de um kit de avaliação baseado no chip Xilinx XC3S250E da família Spartan para o desenvolvimento de hardware baseado na linguajes VHDL ou Veriloj, adicionalmente se desenvolvera o programador da FPGA baseado no chip FTDI ft2232d usando o protocolo JTAG

Hardwrae

O chip XC3S250E da família Spartan pode ser programado como modo maestro usando uma memória externa flash PROM onde estão armazenado a configuração ou código hardware, com este método a FPGA pode se programar por se mesma; porem a FPGA também pode ser programado por um microcontrolador ou microprocessador em modo escravo, usando o porto serial, paralelo ou o protocolo JTAG.

Para o nosso caso, a FPGA será programado em modo escravo pelo protocolo JTAG usando o programador desenvolvido junto a este projeto ou por qualquer microprocessador que use linux como sistema operativo, baseado no programa de software libre xc3sprog, adicionalmente também pudera ser programado por se mesmo usando uma memoria flash de modo que o código seja armazenado dentro da memoria e se programe a se mesmo quando seja energizado.

O kit de avalização contara com as seguintes características::

  • 2 Push Buttom

  • 1 Memoria sdram

  • 5 LEDs

  • 5 Switch

  • 50 Pins de entrada e saída

O programador da FPGA esta composto pelo chip ft2232d, este chip possui dois canais de saída que podem ser programadas como USB -> JTAG, USB->I2C, USB->SPI ou USB->UART cada um, esta caraterística é ideal para programar no só a FPGA sem não também microcontroladores, também é ideal para comunicar um computador com uma memória, um conversor ADC ou DCA, este programador é ideal para projetos de IOT.

Para programar a FPGA, se usara um canal de modo USB->JTAG.

Veículo aéreo não tripulado

(2015-2017)

Objetivo

Analise, desenho e construção de um veículo aéreo não tripulado de baixo custo, desenvolvido com ferramentas de software livre para o apoio em tarefas de aerofotogrametria.

O projeto consiste da analise, desenho, construção e posta em marcha de um veiculo aéreo não tripulado, iniciando desde a toma dos requerimentos ate o produto elaborado.

Para o desenvolvimento do projeto não se tem datas especificas estabelecidas por conta de fatores externos que impedem o cumprimento, porem para a culminação do projeto tem se estabelecido três etapas. A primeira etapa consiste no modela mento matemático e realização física de um protótipo, depois de avaliar o modelo matemático com respeito ao comportamento físico a segunda etapa consiste no analise e desenho se um sistema de controle, seguimento de trajetórias e reconhecimento de imagens de enteres na aerofotogrametria, e na terceira e ultima etapa consiste em embracr o software desenvolvido na segunda etapa em um single board computer (SBC) para ser testado e avaliado no protótipo real desenvolvido na primeira etapa.

Analise

Como anteriormente dito, o avião esta voltado para o uso em aerofotogrametria por tal motivo as condições de velocidade são aproximadamente 20m/s, com uma carga máxima de 3.5kg, controlável e com caraterísticas de estabilidade e com alta capacidade de sustentação.

Existem varias configurações de desenho estrutural do aeronave que devemos ter em conta para lograr os objetivos de requerimentos desejados, as principais configurações estão relacionados como a posição das assas com respeito ao fuselagem (assa alta, assa baixa e assa media), também devemos ter em conta a configuração estrutural do fuselagem (delta ou convencional) ou o desenho da assa (delta ou convencional). Os aviões de combate em sua maioria são manobráveis é dizer são aeronaves que por sua natureza devem gerar movimentos complexos que evitem uma possível colisão com um míssil por isso estas aeronaves tem caraterísticas de desenho como assa delta, assa baixa e um fuselagem em configuração delta que cumpram e pouca sustentação e maior manobrabilidade, porém existem os aviões convencionais que procuram mas o conforto do seus passageiros ou para realizar algum fim que não precise movimentos complexos como a fotogrametria aérea, este tipo de aviões têm uma configuração de assa alta, assa convencional e fuselagem convencional que procurem a sustentação desejada para manter a função desejada.

Como anotado anteriormente a nossa configuração mais recomendável para o nosso requisito principal é a configuração convencional, por tal motivo na tabela baixo podemos ver a configuração de alguns aviões não tripulados que poderíamos ter em conta para o nosso próprio desenvolvimento.

Baseado na tabela anterior, algumas especificações que vamos ter em conta para o desenho estrutural de nosso aeromodelo são as seguintes:

  • envergadura = 1.30 m.
  • largo del fuselaje = 1 m.
  • altura = 0.3 m.
  • velocidad = 20 m/s.
  • Peso = 3.5 kg.

Passada a etapa do desenho estrutural da aeronave, o processo seguinte é o desenho das assas, para o desenho das assa é preciso ter em conta o perfil e a corda (anchura) da assa o perfil alar, com relação na assa principal este deve ser um perfil que gere a sustentação suficiente para manter no ar o aeromodelo, com relação na assa traseira é preciso que o perfil seja mais manobrável pelo que o perfil seja simétrico.

Objetivo

Desarrollo de un software de escritorio y prototipo funcional hardware/software para el registro de tiempos de procesos laborales, voltado a la industria textil

  1. Hardware/software

    El hardwrae que se debe desarollar debe contener un lector rfid donde esta almacenados los datos del trabajador, de tal modo que este al pasar la tarjeta envie una senal al software de escritorio en el que indique el inicio del processo y su posterior parada del processo, tambien debe contar con un boton el cual registre cualquier anomalia p interrupcion en el proceso de tal manera que el softeare de escritorio detecte la interrupcion y automaticamente para el reloj, de igual amnera sucede cuando es accionado de neuvo el boton, este retorna en el contaje del tiempo.

    Dada la simplicidad de arduino, para el diseno del prototipo, se tuvo en cuenta el diseno placa de evaluacon de arduino uno para la creacion del software y luego ser embarcado en el chip de arduino atm328, se tuvo envuenta varias solcuiones para la transmision de los datos entre ellas bluetooth, lora y xbee, sin embago seobt por usar xbee dado su alance y su ancho de banda. el tarjeta que recive los datos es rfid ntc545

    El hardwrae consta de dos modulos, un que recibe la nformacion del emplado y otro es un modulo receptor que recib e los datos de los modulos esclavos y envia esta infromacion por comunicacion serial al software de escritorio.

  2. Software de escritorio

    El software de escritoiro se desarrollo vbajoe el IDE de netbeans con una base de datos posedgre, el objetivo del software de escritorio es presentar la nformacion de los tiempos del operario y ser almacenado en la base de datos, el software automaticamente detecta la interrupcion

El software de escritorio recibe las senales enviadas por el modulo esclavo siendo estas inicio, parada e interrupcion.

Single Board Computer (SBC)

2021-present

Objetivo

El objetivo de este proyecto es disenar y crear un kit de evaluacion basado en tres modulos

  1. Modulo principal

    El mudulo proncipal continene el prceosador IMX6QP qaud core que tiene una memoria eMMC donde almacena el sistema operativo y cuatro memorias DDR3, este modulo consiste de 160 pines de entrada salida que se comunica con los perfericos asociados al carrier board.

  2. Carrier Board

    El Carrier board es el modulo que contiene todos los perifericos asociados al modulo principal, estos perifericos son:

    1. USB
    2. I2C
  3. Modulo DSP

    POr ultimo esta el modulo DSP que contiene una FPGA y modilos adicionales para alamcenar el sistema operativo, se comunica con el carrier board mediante el protocolo PCIe, este modulo puede ser usado para el procesamiento de imagenes dada su alta velocidad de computo
