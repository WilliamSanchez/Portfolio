FPGA

(2013)

Objetivo

Desenvolvimento de um kit de avaliação baseado no chip Xilinx XC3S250E da família Spartan para a criação de codigo hardware baseado na linguajes VHDL ou Veriloj, adicionalmente se desenvolvera o programador da FPGA baseado no chip FTDI ft2232d usando o protocolo JTAG

Hardwrae

O chip XC3S250E da família Spartan pode ser programado como modo maestro usando uma memória externa flash PROM onde estão armazenado a configuração ou código hardware, com este método a FPGA pode se programar por ele mesmo; porem a FPGA também pode ser programado por um microcontrolador ou microprocessador em modo escravo, usando o porto serial, paralelo ou o protocolo JTAG.

Para o nosso caso, a FPGA será programado em modo escravo pelo protocolo JTAG usando o programador desenvolvido junto a este projeto ou por qualquer microprocessador que use linux como sistema operativo, baseado no programa de software libre xc3sprog; também pudera ser programado por se mesmo usando uma memoria flash, de modo que o código seja armazenado dentro da memoria e se programe a se mesmo quando ocorra um corte de energia ou um reset.

O kit de avalização contara com as seguintes características::

  • 2 Push Buttom

  • 1 Memoria sdram

  • 5 LEDs

  • 5 Switch

  • 50 Pins de entrada e saída

O programador da FPGA esta composto pelo chip ft2232d, este chip possui dois canais de saída que podem ser programadas como USB -> JTAG, USB->I2C, USB->SPI ou USB->UART cada um, esta caraterística é ideal para programar no só a FPGA sem não também microcontroladores, também é ideal para comunicar um computador com uma memória, um conversor ADC ou DCA ou qualquer outro dispositivo electrónico que conte com os anteriores protocolos de comunicação; este programador é ideal para projetos de IOT.

Para programar a FPGA, se usara um canal de modo USB->JTAG.

Veículo aéreo não tripulado

(2015-2017)

Objetivo

Analise, desenho e construção de um veículo aéreo não tripulado de baixo custo, desenvolvido com ferramentas de software livre para o apoio em tarefas de aerofotogrametria.

O projeto consiste de analise, desenho, construção e posta em marcha de um veiculo aéreo não tripulado, iniciando desde a toma dos requerimentos ate o produto elaborado.

Para o desenvolvimento do projeto não se tem datas especificas estabelecidas por conta de fatores externos que impedem o cumprimento, porem para a culminação do projeto tem se estabelecido três etapas. A primeira etapa consiste no modelamento matemático e realização física de um protótipo, depois de avaliar o modelo matemático com respeito ao comportamento físico a segunda etapa consiste no analise e desenho se um sistema de controle, seguimento de trajetórias e reconhecimento de imagens de enteres na aerofotogrametria, e na terceira e ultima etapa consiste em embracr o software desenvolvido na segunda etapa em um single board computer (SBC) para ser testado e avaliado no protótipo real desenvolvido na primeira etapa.

Analise

Como anteriormente dito, o avião esta voltado para o uso em aerofotogrametria por tal motivo as condições de velocidade são aproximadamente 20m/s, com uma carga máxima de 3.5kg, controlável e com caraterísticas de estabilidade e com alta capacidade de sustentação.

Existem varias configurações de desenho estrutural do aeronave que devemos ter em conta para lograr os objetivos de requerimentos desejados, as principais configurações estão relacionados como a posição das assas com respeito ao fuselagem (assa alta, assa baixa e assa media), também devemos ter em conta a configuração estrutural do fuselagem (delta ou convencional) ou o desenho da assa (delta ou convencional). Os aviões de combate em sua maioria são manobráveis é dizer são aeronaves que por sua natureza devem gerar movimentos complexos que evitem uma possível colisão com um míssil por isso estas aeronaves tem caraterísticas de desenho como assa delta, assa baixa e um fuselagem em configuração delta que cumpram e pouca sustentação e maior manobrabilidade, porém existem os aviões convencionais que procuram mas o conforto do seus passageiros ou para realizar algum fim que não precise movimentos complexos como a fotogrametria aérea, este tipo de aviões têm uma configuração de assa alta e convencional, a fuselagem também é convencional de tal modo que procurem a sustentação desejada para manter a missão estabelecida.

Como anotado anteriormente a nossa configuração mais recomendável para o nosso requisito principal é a configuração convencional, por tal motivo na tabela baixo podemos ver a configuração de alguns aviões não tripulados que poderíamos ter em conta para o nosso próprio desenvolvimento.

Baseado na tabela anterior, algumas especificações que vamos ter em conta para o desenho estrutural de nosso aeromodelo são as seguintes:

  • envergadura = 1.30 m.
  • largo del fuselaje = 1 m.
  • altura = 0.3 m.
  • velocidad = 20 m/s.
  • Peso = 3.5 kg.

Passada a etapa do desenho estrutural da aeronave, o processo seguinte é o desenho das assas, para o desenho das assa é preciso ter em conta o perfil e a corda (anchura) da assa o perfil alar, com relação na assa principal este deve ser um perfil que gere a sustentação suficiente para manter no ar o aeromodelo, com relação na assa traseira é preciso que o perfil seja mais manobrável pelo que o perfil seja simétrico.

Baseado na informação anterior, e usando o software libre XFLR5 se pode analizar o comportamento do perfil em diferentes angulso de ataque, para a assa principal os perfiles alares que foram escolhidos são 4412, 4415, 8414 e S9000 na figura abaixo se observa el comportamento de cada um dos perfiles alares com respeito ao coeficiente de arrastre, coeficiente de sustentação e o coeficiente de momentum. Na figura se observa que o perfil alar 4415 tem menos coeficiente de arrastre e uma bom coeficiente de sustentação alem de um coeficiente de momento menor, por tal motivo este perfil é o escolhido para o desenho da assa principal.

Simulação

Antes de desenvolver fisicamente o aeromodelo é preciso simular os parâmetros de desenho estabelecidos nas etapas anteriores como o objetivo de fazer mudanças antes de desenvolve-lo fisicamente.

Existem vários software de simulação de vôos como Microsoft flight simulator, x-plane, emtre outros, esses softwares são licenciados e possuem excelente qualidade de realismo, porem ao ser software de caráter propriativo não é possível fazer alterações acorde com as nossas necessidades, alem de ser muito custoso. Por tal motivo é usado flightegar já que ao ser um software libre pode-se realizar análises acorde a nossas necessidades; para a construção do aeromodelo virtual, é preciso usar o sfotware blender para pode-lo usar no simulador.

CETCO

[2018-2019]

Objetivo

Desenvolvimento de um protótipo funcional hardware/software e um programa de escritório para o registro de tempos nos processos laborais voltados para a indústria têxtil

Hardware/Firmware

O hardware há desenvolver contem um leitor RFID onde envie informações ao microcontrolador quando um funcionário passa uma tarjeta, este protótipo envia uma sinal ao software de escritório em que indique o início do processo e a sua posterior finalização, também deve conter um button switch no qual registre qualquer anomalia ou interrupção no processo laboral de um produto, de modo que o software de escritório detecte a interrupção automaticamente e pare o relógio, também é preciso que quando se acione de novo o buton switch esta sinal é enviada ao software de escritório para retornar a contagem do relógio. O software de escritório deve armazenar os dados obtidos com o envio dos sinais.

Para o desenvolvimento do hardware do protótipo se teve em conta o desenho da placa rfid pn532 e para o desenvolvimento do firmware se usou a linguajem C++ para logo ser embarcado em um chip atmega328, se teve em conta varias tecnologias de comunicação para a transmissão dos dados entre o modulo escravo e o modulo receptor conectado ao software de escritório, estas tecnologias foram bluetooth, LoRa e XBEE, porem se decidiu usar xbee já que a seu rango e ancho de banda cumprem as condições estabelecidas.

O protótipo funcional consta de dois módulos, um módulo escravo que recebe a informação dos sinais do funcionário ao passar a tarjeta ou pressionar o button switch e o outro módulo receptor que recebe os dados dos módulos escravos e envia esta informação por comunicação serial ao software de escritório.

Software de Escritorio

O software de escritório é desenvolvido usando o IDE de Netbeans, este software de escritório contem uma base de dados MySQL, a função de este software é a apresentar a informação dos tempos do operário e são armazenados na base de dados.

O software de escritório recebe os sinais enviadas pelo módulo escravo sendo estas o início, parada e interrupção.

Conclusão

É preciso trocar o chip atmega328 por uma da família stm,32 de baixa potencia,

Melhorar o software de escritório, aumentando a funcionalidade do software de escritório além de armazenar os dados provenientes dos funcionários, também é preciso que o software de escritório calcule o promedio de cada um dos processos, cálculo de tempo total na realização da peça em estudo, entre outras funções.

Single Board Computer (SBC)

2021-present

Objetivo

O objetivo de este projeto é desenhar e desenvolver um kit de avaliação baseado em três módulos: um módulo principal que possui o processador, um módulo carrier board que contem todos os periféricos associados ao processador, e por ultimo um módulo que contem uma DSP para o processamento de imagens.

  1. Modulo principal

    O modulo principal contem o processador IMX6QP quad core que tem uma memória eMMC onde armazena o sistema operativo e quatro memorias DDR3, este modulo consiste de 160 pines de entrada e saída que comunica-se com os periféricos associados ao Carrier board.

  2. Carrier Board

    o carrier board é o modulo que contem todos os periféricos associados ao modulo principal, estos periféricos são:

    1. USB
    2. I2C
    3. mPCIe
    4. HDMI
    5. LVDs
    6. GPIO
    7. Audio
    8. Micrcontrolador
  3. Modulo DSP

    O terceiro módulo, contem um chip DSP, uma PGA e módulos adicionais para armazenar o sistema operativo, se comunica com o carrier boaras mediante o protocolo mPCIe. Este módulo pode ser usado para o processamento de imagens por conta de alta velocidade de cômputo.
